# QuanMock

## 🎯 核心解决场景

**痛点假设**：
你需要测试一个特定的前端样式——当用户年龄**超过 100 岁**时，页面字体需要加粗显示。但在真实的测试系统或数据库中，很难找到符合该极端条件的客户数据，导致样式测试难以推进。

**QuanMock 的解决方案**：
利用本项目，你可以直接拦截网络请求并**重写接口返回的数据**。只需将响应 JSON 中的年龄字段轻松修改为你需要的数值（如 `101`），即可零成本完成前端样式的验证，告别繁琐的底层数据伪造。

---

## 🚀 待解决的问题 (Roadmap)

计划在未来的版本中实现以下功能扩展：

1. **适配非 JSON 的响应**：打破仅支持 JSON 的限制，兼容 XML、纯文本等更多数据格式的处理。
2. **对请求体进行修改**：不仅能重写返回结果 (Response)，还能在请求发送前拦截并修改请求参数 (Request Body)。
3. **加入 AI 造数据**：接入 AI 能力，根据接口结构自动生成符合逻辑和业务场景的 Mock 数据，彻底解放双手。
